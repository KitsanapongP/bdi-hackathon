# Hackathon Platform System Flow

เอกสารนี้อธิบาย Flow การทำงานของระบบทั้งหมดแบบเป็นขั้นเป็นตอน (Step-by-step) โดยอ้างอิงจาก Source Code ในปัจจุบัน (Frontend & Backend) เพื่อให้สามารถนำไปใช้วางแผนงานและตรวจสอบสิ่งที่ต้องทำต่อ (To-Do/Plan) ได้

---

## 🚀 1. ฝั่งผู้ใช้งานทั่วไป (User Flow)

### 1.1 การสมัครสมาชิกและเข้าสู่ระบบ (Registration & Auth)
- **Sign Up:** ผู้ใช้เข้ามาหน้า `/home/register` กรอกข้อมูลส่วนตัว (ชื่อ-นามสกุล TH/EN, เพศ, วันเกิด, ระดับการศึกษา, ภูมิลำเนา, สถาบัน) ข้อมูลติดต่อ (อีเมล, เบอร์โทรศัพท์) และบัญชี (Username, Password)
- **Consent:** ก่อนกดสมัคร **ต้องผ่านการยินยอม (Consent)** ตามเอกสารข้อกำหนดการใช้งาน (`TERMS`) และ นโยบายส่วนบุคคล (`PRIVACY/PDPA`) ซึ่งระบบจะดึงจาก API อัตโนมัติ หากไม่ยอมรับจะไม่สามารถสมัครได้
- **Log In:** เข้าสู่ระบบด้วย Email + Password ระบบจะจ่าย JWT และบันทึก Cookie/localStorage

### 1.2 ล็อบบี้การจัดการทีม (Team Lobby)
เมื่อผู้ใช้งานเข้าสู่ระบบและเข้าไปที่หน้าทีม (`/home/team`) หาก**ยังไม่มีทีม** จะเข้าสู่หน้า Lobby ซึ่งมี 4 ตัวเลือกหลัก:
1. **สร้างทีม (Create Team):** สร้างทีมใหม่ กำหนดชื่อทีม และเลือกว่าเป็น Public (ค้นหาและขอเข้าได้) หรือ Private
2. **เข้าทีมด้วยรหัส (Join by Code):** กรอกรหัส Invite Code 6 หลักที่ได้จากหัวหน้าทีม Private เพื่อส่งคำขอเข้าร่วม
3. **ค้นหาทีมสาธารณะ (Browse Public Teams):** ดูรายชื่อทีม Public ที่ยังมีสมาชิกไม่เต็ม (max 5 คน) และกดขอเข้าร่วม (Join Request)
4. **จัดการคำเชิญ (Invitations):** หากมีหัวหน้าทีมพิมพ์ Username เชิญเราเข้าทีม เราสามารถกดยอมรับ/ปฏิเสธ ได้ที่นี่

### 1.3 การจัดการและบริหารทีม (Team Management)
หากผู้ใช้งานมีทีมแล้ว (ไม่ว่าจะเป็นหัวหน้า หรือสมาชิก) จะเห็นหน้า Dashboard ของทีม
- **สมาชิก (Members):** 1 ทีมมีสมาชิกได้สูงสุด 5 คน
- **สิทธิ์หัวหน้าทีม (Leader):** 
  - อนุมัติ/ปฏิเสธ คนที่ขอเข้าร่วมทีม
  - ใช้ Username และส่งคำเชิญ (Invite) ให้คนอื่น
  - เตะสมาชิกออกจากทีม
  - โอนสิทธิ์หัวหน้าทีมให้คนอื่น
  - ยุบทีม (Disband Team)
- **สิทธิ์สมาชิก (Member):**
  - กดออกจากทีม (Leave Team) ด้วยตัวเองได้

### 1.4 การยืนยันตัวตนเอกสาร (Identity Verification)
เพื่อรับสิทธิ์เข้าแข่งขัน แต่ละทีมต้องทำการส่งเอกสารและยืนยันตัวตน
- **Upload Document:** สมาชิกแต่ละคนจะต้องอัปโหลดไฟล์/เอกสารยืนยันตัวตนของตนเองในหน้าทีม (อัปโหลดเพิ่ม ลบ แก้ไขชื่อ ได้)
- **Self-Confirm:** เมื่อเอกสารครบ สมาชิกแต่ละคน **ต้องกดยืนยัน (Confirm)** เอกสารของตัวเอง
- **Team Submit:** เมื่อสมาชิก**ทุกคน**ในทีมกดยืนยันตัวเองเรียบร้อย **หัวหน้าทีม** จะมีปุ่ม "ส่งทีมเพื่อยืนยัน" (Submit Team) ปรากฏขึ้น พอกดแล้ว สถานะทีมจะเข้าสู่ขั้นตอนส่งตรวจและไม่สามารถแก้อะไรในทีมได้อีกจนกว่า Admin จะตรวจ
- *หมายเหตุ: หากมีการยุบทีมไปแล้ว (Disband) สถานะการยืนยันจะถูก Mark ว่า 'cancelled' ในระบบหลังบ้าน*

### 1.5 การส่งผลงานแข่งขัน และที่ปรึกษา (Submission & Advisor)
- **อาจารย์ที่ปรึกษา (Advisor):** หัวหน้าทีมสามารถเพิ่มรายชื่ออาจารย์ที่ปรึกษาได้ โดยระบบจะเช็คระดับ Email ว่าต้องไม่ซ้ำซ้อนกับทีมอื่นๆ
- **การส่งผลงาน (Works):** หัวหน้าทีมสามารถกรอกลิงก์ผลงาน (YouTube / Google Drive) และสามารถแนบไฟล์หลายไฟล์ (PDF, DOCX, PPTX, PNG) ลงไปในระบบได้

## 🛡️ 2. ฝั่งผู้ดูแลระบบ (Admin Flow)

Admin ใช้งานผ่าน Context หน้า `/admin` และมี API รองรับดังนี้:
- **Dashboard Overview:** ดูสถิติรวมของระบบ
- **Manage Allowlist:** ควบคุมรายการที่อนุญาตเป็นพิเศษ
- **Settings & Content:**
  - จัดการของรางวัล (Rewards)
  - จัดการรายชื่อสปอนเซอร์ และ อัปโหลดโลโก้ (Sponsors) สามารถสลับจัดเรียงลำดับได้
  - จัดการข้อมูลการติดต่อและช่องทางติดต่อสื่อสาร (Contacts & Channels)
  - จัดการตารางกิจกรรมกำหนดการ (Schedule)
- **Data Export:** สามารถ Export ข้อมูลของทีมที่ยื่นเอกสารการทำ Verification สำเร็จแล้วออกไปเป็น Bundle (.zip / .csv) ได้

---

ข้างบนคือ Flow ทั้งหมดของระบบในตอนนี้สิ่งที่เราจะเพิ่มเข้าไปจะมีตามนี้
หลังจาก 1.4 เพิ่มระบบหลังจากมีการคัดเลือกแล้ว หลังจากคัดเลือกเสร็จแล้ว จะมีการแจ้งเตือนให้ทุกคนในทีมว่าผ่านหรือไม่ผ่านการคัดเลือก แล้วที่หน้า ยืนยันตัวตน ของ Leader จะมีปุ่ม "ยืนยัน" ให้กดเพื่อเป็นการยืนยันการเข้าร่วมโครงการอีกครั้งนึง (ซึ่งปุ่มนี้จะมีเวลาจำกัดในการกด) แล้วถ้าไม่กดหรือกดไม่ทันในช่วงเวลาที่กำหนด ก็จะนับเป็น "ไม่ผ่าน" แล้วทาง Admin ก็จะไปเลือกทีมอื่นแทน

หากกดยืนยันที่จะเข้าร่วมแล้ว ทุกคนในทีมจะมีเมนูในหน้า Profile เพิ่มมาคือ สิทธิ์ประโยชน์ ซึ่งแต่ละสิทธิ์มีจะมี QR code เป็นของตัวเอง เช่น เสื้อ ค่าอาหาร ค่าที่พัก และอื่น ๆ วิธีใช้ตัว QR code นี้คือในงานจริง เราจะให้คนที่มีสิทธิ์ประโยชน์ ไปยื่นให้ พนักงาน(ที่ใช้ระบบ Admin) เป็นคนแสกน QR code แล้วพนักงานจะส่งข้อมูลนี้ไปยัง Admin ผ่านระบบหลังบ้านว่า สิทธิ์ประโยชน์เสื้อของ user นี้ ได้รับแล้ว

Feature ที่น่าจะต้องเพิ่มตาม flow:
เพิ่มระบบการแจ้งเตือนเมล
- ที่ขั้นตอน 1.4 การยืนยันตัวตนเอกสาร (Identity Verification) หลังจาก Team Submit แล้วจะมีการแจ้งเตือนให้ Admin และ member ทุกคนในทีม ทราบว่ามีทีมยื่นเอกสารแล้ว 
- เพิ่มตอนที่รู้ผลการคัดเลือกแล้วว่าผ่านหรือไม่ผ่าน
- เพิ่มสำหรับทีมที่ผ่านการคัดเลือกแล้ว ที่ขั้นตอน การยืนยันการเข้าร่วมโครงการ ให้มีการแจ้งเตือนให้ Admin และ member ทุกคนในทีมว่ายืนว่าจะเข้าร่วมโครงการ

และในหน้า ทีมของฉัน 
- เพิ่มเมนู กล่องข้อความ สำหรับเก็บ log การแจ้งเตือนเมลที่ระบบส่งไปหา member แต่ละคน

ส่วนการแจ้งเตือน Admin
- เพิ่มการตั้งค่าแจ้งเตือน ว่าจะส่งเมลไปหาแอดมินคนไหน (ระบบนี้จะมี admin หลายคนที่ทำงานคนละส่วนกัน)
